# Contributing

Thank you for your interest in contributing.

## Ways to Contribute

- Fix typos or grammatical mistakes
- Improve explanations
- Add practical examples
- Add interview questions
- Improve diagrams
- Add troubleshooting tips
- Suggest new Git or GitHub topics

---

## Contribution Workflow

1. Fork the repository.
2. Create a new branch.

```bash
git checkout -b feature/improve-documentation
```

3. Make your changes.

4. Commit your work.

```bash
git commit -m "Improve Git Fundamentals documentation"
```

5. Push your branch.

```bash
git push origin feature/improve-documentation
```

6. Open a Pull Request.

---

## Documentation Style

Please follow these guidelines:

- Use Markdown.
- Write clear and concise explanations.
- Explain the purpose of each Git command.
- Include practical examples whenever possible.
- Add DevOps-related use cases when applicable.
- Avoid unnecessary complexity.

---

## Reporting Issues

If you find an error or have a suggestion:

- Open an Issue.
- Clearly describe the problem.
- Include screenshots if relevant.

---

Thank you for helping improve this repository.
